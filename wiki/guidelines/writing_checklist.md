# 글쓰기 검수 체크리스트
_코마인드웍스 LLM Wiki v5.0 — 글 초안 완성 후 반드시 통과_

---

## 필수 체크 (모두 통과해야 발행)

### 훅 (오프닝)
- [ ] 첫 문장이 독자를 멈추게 하는가?
- [ ] "이거 나 얘기다"라고 느낄 수 있는 공감 포인트가 있는가?
- [ ] 클리셰("안녕하세요", "오늘은 ~에 대해 알아보겠습니다") 없이 시작하는가?

### 나다움
- [ ] 내 경험/일기에서 실제 사례가 최소 1개 이상 들어갔는가?
- [ ] 내 생각/관점이 명확히 드러나는가? (남들과 다른 시각)
- [ ] "나"만 쓸 수 있는 이야기가 있는가?

### 감정 곡선
- [ ] 글 전체에 감정의 흐름이 있는가? (문제 → 좌절 → 시도 → 발견 → 통찰)
- [ ] 독자가 끝까지 읽고 싶은 긴장감이 유지되는가?

### 구조
- [ ] 핵심 메시지가 1개로 명확한가? (2개 이상이면 분리)
- [ ] 소제목이 글의 흐름을 안내하는가?
- [ ] 결론이 행동 촉구 또는 인사이트로 끝나는가?

### 정보 품질
- [ ] 팩트는 출처(Wiki 페이지)로 뒷받침되는가?
- [ ] 주장이 구체적인 수치나 사례로 뒷받침되는가?
- [ ] 오해를 살 수 있는 모호한 표현은 없는가?

## 플랫폼별 추가 체크

### 블로그 (브런치/티스토리)
- [ ] 제목: 숫자 또는 구체적 이익 포함 ("3가지", "하는 방법", "실패한 이유")
- [ ] 분량: 1,500~3,000자
- [ ] SEO 키워드: 제목, 첫 문단, 소제목에 자연스럽게 포함

### 유튜브 스크립트
- [ ] 오프닝 15초: 시청자가 떠나지 않게 할 훅
- [ ] 말하는 것처럼 쓰여졌는가? (구어체, 짧은 문장)
- [ ] 화면 전환 포인트가 표시되어 있는가?

### SNS (X/스레드)
- [ ] 280자 이내의 임팩트
- [ ] 해시태그 3개 이내
- [ ] 댓글 유도 질문으로 끝나는가?

## 검수 결과 판정

| 항목 수 | 판정 |
|--------|------|
| 필수 15개 모두 통과 | ✅ 발행 가능 |
| 필수 12~14개 통과 | ⚠️ 수정 후 재검수 |
| 11개 이하 통과 | ❌ 전면 재작성 |
