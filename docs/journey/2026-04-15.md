## 배운 것

- **Claude Code skill 커스터마이즈 패턴**: SKILL.md에 도메인 고정값(n8n, Notion 등)을 명시하면 매 세션마다 컨텍스트를 다시 설명할 필요가 없다. 에이전트가 처음부터 내 환경을 알고 시작한다.
- **오케스트레이터 설계 원칙**: Jarvis 같은 총괄 에이전트는 "두뇌"가 아닌 "라우터"여야 한다. 명령 카탈로그 + 에스컬레이션 규칙 + State on Disk — 이 3요소가 핵심.
- **에스컬레이션 정량화**: "판단이 어려우면 물어봐"를 "의도 분류 신뢰도 70% 미만이면 텔레그램으로 보고"로 바꾸면 에이전트가 실제로 동작할 수 있다.

## 핵심 인사이트

> **프로젝트가 에이전트를 소유한다.**

보통 "AI 에이전트를 만들어야지"라고 생각하면 에이전트 중심으로 설계한다. 하지만 실제로 돌아가게 하려면 반대로 생각해야 한다 — 각 프로젝트 폴더가 자기 에이전트를 소유하고, 공통 스킬은 상위에서 공유한다. 이렇게 하면 Claude Code가 실행 폴더 기준으로 해당 에이전트만 자동으로 로드한다.

> **비용 최적화는 레이어 분리에서 나온다.**

n8n(비용 없음) → Claude Code(비용 없음) → OpenClaw(heartbeat 최소화) → Claude API(복잡한 것만) 순서로 레이어를 나누면, 대부분의 자동화는 API 비용 없이 돌아간다.

## 유튜브 대본 포인트

- **"AI 비서를 만드는 게 아니라 AI 팀을 조직한다"** — 단일 챗봇 대신 역할별 에이전트를 설계하는 개념 전환
- **Claude Code vs Claude.ai 역할 분담** — 결과물이 파일/코드면 Claude Code, 생각/판단이면 Claude.ai. 이 기준 하나로 혼란이 사라진다
- **비용 0원으로 24시간 AI 비서 돌리기** — n8n + Claude Code + OpenClaw 레이어 구조 실제 시연
- **메타 스킬이란 무엇인가** — blueprint → deep-dive → 구현 → autoresearch → reflect 워크플로우를 처음 접하는 사람 눈높이로 설명
