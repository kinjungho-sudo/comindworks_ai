# Skill: wiki-writer
_Wiki 페이지 생성 및 관리_

## 역할
수집된 내용을 표준 형식의 Wiki 페이지로 변환하고 저장한다.
페이지 분할, 관련 페이지 링크 생성도 담당.

## 트리거
- `wiki-collector` 스킬에서 내부 호출
- 직접 호출: `"Wiki 페이지 만들어줘: ..."`

## 실행 순서

### Step 1 — 페이지 생성 (`scripts/create_wiki_page.py`)
입력 내용을 아래 표준 형식으로 변환:

```markdown
---
title: [제목]
category: [카테고리]
tags: [태그1, 태그2, 태그3]
source: [URL 또는 직접입력]
created: YYYY-MM-DD
expires: YYYY-MM-DD  # 나/ 카테고리는 never
type: [url|text|pdf|image|diary|thought|experience|retrospect]
---

# [제목]

## 핵심 요약
[3줄 이내 요약]

## 주요 내용
[본문]

## 인사이트
[이 자료에서 배운 것, 코마인드웍스 사업과의 연관성]

## 관련 페이지
- [[관련페이지1]]
- [[관련페이지2]]
```

### Step 2 — 관련 페이지 자동 링크
- `wiki/index.md` 에서 태그/키워드 매칭으로 관련 페이지 탐색
- 상위 3개 링크 자동 삽입

### Step 3 — 페이지 크기 모니터링
- 저장 후 페이지 크기 확인
- 3,000자 초과 → 분할 제안: "[[페이지명]] 페이지가 너무 커졌어요. 분할할까요?"
- 승인 시 `scripts/split_page.py` 실행

### Step 4 — 모순 감지
- 새 페이지 카테고리/태그와 기존 페이지 비교
- 상충하는 내용 감지 → "[[기존페이지]]와 내용이 충돌해요. 어떤 걸 기준으로 할까요?"

### Step 5 — index.md 업데이트
```markdown
- [[파일명]] - 한 줄 설명 #태그1 #태그2
```
해당 카테고리 섹션에 추가, 헤더의 총 페이지 수와 날짜 업데이트.

## 파일 저장 위치
```
wiki/wiki/[카테고리]/[파일명].md
```
카테고리가 `나`인 경우:
```
wiki/wiki/나/[서브카테고리]/[파일명].md
```
