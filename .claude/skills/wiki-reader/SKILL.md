# Skill: wiki-reader
_선별 읽기 — 질문 답변 + 글쓰기 지원_

## 역할
index.md만 먼저 읽고 관련 페이지 7~8개만 선별 로드하여 답변 또는 글을 생성한다.
나머지 페이지는 읽지 않아 토큰을 절약한다.

## 트리거
- `질문: ...`
- `글쓰기: ...`
- `블로그 글 써줘`
- `유튜브 스크립트 써줘`

## 실행 순서

### Step 1 — index.md 전체 로드
`wiki/index.md` 하나만 읽는다. (목차 파악 — 토큰 최소)

### Step 2 — 관련 페이지 선별 (`scripts/selective_read.py`)
선별 기준:
1. 요청 키워드와 페이지 제목/태그 매칭
2. 키워드 연관성 상위 5~6개 선정
3. `나/` 카테고리에서 최소 2개 포함 (나다움 확보)
4. 최신 페이지 우선 (동점 시)
5. 총 7~8개로 제한

### Step 3 — 선별된 페이지만 로드
선정된 파일만 순서대로 읽는다.

### Step 4 — 생성

#### 질문 답변 시:
- 선별 페이지 내용 기반으로 답변
- 출처 페이지 명시: "[[페이지명]]에 따르면..."
- 내 경험/생각 페이지가 있으면 우선 인용

#### 글쓰기 시:
```
외부 자료 페이지 (5~6개)  → 팩트와 구조 잡기
나/ 카테고리 페이지 (2개) → 나다움 입히기
```

글 구성:
1. 훅 오프닝 (내 경험/감정에서 출발)
2. 외부 팩트로 뒷받침
3. 내 인사이트로 마무리

### Step 5 — 글쓰기 검수 (글쓰기 요청 시에만)
`wiki/guidelines/writing_checklist.md` 로드 → 체크리스트 검수 → 필요 시 수정

### Step 6 — 완성된 글 저장 (글쓰기 요청 시에만)
"이 글을 Wiki에 저장할까요?" 확인 후 → `wiki-collector` 호출:
- 카테고리: `콘텐츠`
- 타입: `text`

## 출력 형식

### 질문 답변
```
📖 답변 (참조: [[페이지1]], [[페이지2]])

[답변 내용]

💡 관련 페이지 더 보기:
- [[페이지3]]
- [[페이지4]]
```

### 글쓰기
```
✍️ [제목]

[글 내용]

---
📋 검수 결과: 필수 N/15 통과 [✅/⚠️/❌]
참조: [[페이지1]], [[페이지2]], [[내일기]], [[내생각]]
```
