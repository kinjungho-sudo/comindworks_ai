# Skill: wiki-search-collect
_주제 기반 웹 검색 → 콘텐츠 수집 → Wiki 저장 → Git push 자동화_

## 역할
사용자가 주제와 개수를 입력하면, 웹에서 관련 고품질 아티클을 찾아 Wiki 페이지로 저장한다.

## 트리거
- `wiki-search-collect [주제] [개수]`
- 또는 인수(ARGUMENTS)로 전달

## 실행 순서

### Step 1 — 입력 파싱
ARGUMENTS에서 주제와 개수를 파싱한다.
- 형식: `[주제] [개수]` (예: "1인 IT 창업 5")
- 개수 기본값: 5
- 주제가 없으면 사용자에게 요청: "어떤 주제로 몇 개 수집할까요?"

### Step 2 — 현황 파악
`wiki/index.md`를 읽어 이미 저장된 관련 페이지 확인 (중복 방지)

### Step 3 — 웹 검색 (병렬)
주제 관련 검색어 2~3개를 병렬로 WebSearch 실행:
- 검색어 1: `[주제] 인사이트 블로그 2025 2026`
- 검색어 2: `[주제] 성공사례 전략 핵심`
- 검색어 3: `[주제] site:brunch.co.kr OR site:maily.so OR site:eopla.net`

### Step 4 — URL 선별
검색 결과에서 아래 기준으로 N개 URL 선별:
- 구체적 수치/사례 포함 여부
- 코마인드웍스 사업(AI·자동화·창업)과 연관성
- 페이월/로그인 필요 URL 제외
- 이미 저장된 페이지와 중복 제외

### Step 5 — 콘텐츠 수집 (병렬)
선별된 URL을 WebFetch로 병렬 수집:
- 각 URL에 프롬프트: "핵심 인사이트, 구체적 수치, 전략을 한국어로 상세히 추출해줘"
- 수집 실패 시 대체 URL로 교체

### Step 6 — Wiki 페이지 생성 (병렬)
수집된 내용으로 Wiki 페이지 N개 동시 생성:

**파일 경로**: `wiki/wiki/[카테고리]/[파일명].md`
**파일명 규칙**: `[카테고리약어]-[핵심키워드]-[YYYY-MM-DD].md`

**페이지 형식**:
```markdown
---
title: 페이지 제목
category: 카테고리명
tags: [태그1, 태그2]
source: URL
created: YYYY-MM-DD
expires: YYYY-MM-DD
type: url
---

# 제목

## 핵심 요약
3줄 이내

## 주요 내용
본문

## 인사이트
코마인드웍스 관점에서 배운 것

## 관련 페이지
- [[링크1]]
```

**카테고리 판단 기준**:
| 카테고리 | 키워드 |
|---------|-------|
| AI에이전트 | 에이전트, Claude, GPT, LLM, 프롬프트 |
| 자동화 | n8n, Make, 워크플로우, 자동화 |
| 창업 | 비즈니스, 창업, 수익화, SaaS, 스타트업 |
| 콘텐츠 | 유튜브, 블로그, SNS, 콘텐츠 |
| 개발 | Python, API, 코딩, 개발 |
| 리서치 | 논문, 트렌드, 시장분석 |

### Step 7 — index.md & log.md 업데이트
- `wiki/index.md`: 카테고리별 항목 추가, 총 페이지 수 업데이트
- `wiki/log.md`: 이력 추가

### Step 8 — Git push
```bash
git add wiki/
git commit -m "wiki: [카테고리] — [주제] 인사이트 N편"
git push origin master
```

### Step 9 — 완료 보고
```
✅ [N]개 저장 완료 → Obsidian 5분 내 자동 반영

| # | 제목 | 핵심 한 줄 |
|---|------|----------|
| 1 | ... | ... |
...

📂 카테고리: [카테고리]
🔗 출처: 웹검색 수집
```

## 주의사항
- 페이월/로그인 필요 URL은 스킵하고 대체 URL 사용
- 수집 실패 시 log.md에 ERROR 기록
- 이미 유사 페이지가 있으면 사용자에게 알리고 계속 진행 여부 확인
- 유통기한: 기본 생성일 + 6개월 (나/ 카테고리는 never)
