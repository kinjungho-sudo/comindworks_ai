# Skill: wiki-inspector
_내부 관리 + 정기 점검_

## 역할
Wiki의 건강 상태를 점검한다.
만료 페이지, 모순 감지, 페이지 크기 초과, 카테고리 불균형 등을 리포트하고 정리한다.

## 트리거
- `Wiki 점검`
- `점검해줘`
- `만료 확인`
- `모순 확인`
- 주간 자동 점검 (매주 월요일 대화 시작 시)

## 실행 순서

### Step 1 — 현황 수집 (`scripts/inspect_wiki.py`)
- `wiki/index.md` 전체 파싱
- 총 페이지 수, 카테고리별 분포 집계
- 이번 주 추가 페이지 목록 (log.md에서)

### Step 2 — 만료 점검 (`scripts/expiry_check.py`)
- 모든 Wiki 페이지의 `expires:` frontmatter 읽기
- 오늘 날짜 기준 만료된 페이지 목록 추출
- 30일 내 만료 예정 페이지 목록 추출
- 나/ 카테고리는 제외 (유통기한 없음)

### Step 3 — 모순 감지 (`scripts/conflict_check.py`)
- 동일 키워드를 가진 페이지들 간 상충 내용 탐지
- (예: "n8n은 유료다" vs "n8n은 무료다")
- 감지 시 → 사용자에게 어떤 걸 기준으로 할지 질문

### Step 4 — 크기 점검
- 3,000자 초과 페이지 목록 추출
- 분할 제안 대기

### Step 5 — 점검 리포트 출력

```
🔍 Wiki 점검 리포트 — YYYY-MM-DD

📊 현황
  총 페이지: N개
  카테고리 분포:
  - AI에이전트: N개
  - 자동화: N개
  - 창업: N개
  - 콘텐츠: N개
  - 개발: N개
  - 리서치: N개
  - 나: N개
  - 기타: N개

⏰ 만료 현황
  이미 만료: N개
  → [페이지명1], [페이지명2]
  30일 내 만료 예정: N개
  → [페이지명3]

⚠️ 모순 감지: N건
  → [[페이지A]] vs [[페이지B]]: [충돌 내용 요약]

📏 크기 초과: N개
  → [[페이지명]]: 3,450자 (분할 권장)

✅ 권장 조치
  1. [조치1]
  2. [조치2]
```

### Step 6 — 자동 조치 (승인 시)
- 만료 페이지: `wiki/wiki/기타/archived/` 로 이동 + index.md에서 제거
- 크기 초과: 분할 실행
- 모순: 사용자가 선택한 버전 유지, 나머지 deprecated 태그

## 점검 주기
- 수동: 언제든지 트리거 가능
- 자동: 매주 월요일 대화 시작 시 간단 현황 보고
