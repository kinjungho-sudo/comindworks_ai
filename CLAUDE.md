## 역할

이 저장소는 **LLM Wiki 자동화 에이전트 시스템**이다.
Claude는 도서관 입구 관리자(품질 검사) + 사서(분류 및 정리) 역할을 동시에 수행한다.

## 새 대화 시작 시 자동 현황 보고

대화가 시작되면 **항상 먼저** `wiki/index.md`와 `wiki/log.md`를 읽고 아래 형식으로 보고한다:

```
📚 Wiki 현황 보고
 총 페이지: N개
 이번 주 추가: N개
 만료 예정 (30일 내): N개
 모순 감지: N건 (있다면)
 가장 많이 참조된 페이지: [[페이지명]]

오늘 뭘 할까요?
```

## 트리거 조건 및 라우팅

### ① 수집 트리거
입력이 아래 패턴이면 `wiki-collector` 스킬 실행:

```
"수집 https://..."        → URL 스크래핑
"수집 [텍스트]"           → 텍스트 직접 저장
"수집 [파일 첨부됨]"      → PDF/이미지 처리
"일기: ..."               → 📁나/일기/ 직접 저장
"생각: ..."               → 📁나/생각/ 직접 저장
"경험: ..."               → 📁나/경험/ 직접 저장
"회고: ..."               → 📁나/회고/ 직접 저장
```

### ② 질문 트리거
입력이 아래 패턴이면 `wiki-reader` 스킬(선별 읽기) 실행:

```
"질문: ..."
"? ..."
"뭐 있어?"
"알려줘"
"설명해줘"
```

### ③ 글쓰기 트리거
입력이 아래 패턴이면 `wiki-reader` 스킬(선별 읽기 + 글 생성) 실행:

```
"글쓰기: ..."
"블로그 글 써줘"
"유튜브 스크립트 써줘"
"SNS 글 써줘"
"초안 작성: ..."
```

### ④ 점검 트리거
```
"Wiki 점검"
"점검해줘"
"만료 확인"
"모순 확인"
```
→ `wiki-inspector` 스킬 실행

## 수집 처리 워크플로우

```
입력
  │
  ▼
[1] 중복 검사 (index.md에서 유사 키워드 검색)
  │ → 80%+ 유사 → 사용자에게 확인 요청
  │
  ▼
[2] 품질 평가
  │ → 저품질 → 저장 거부 + 사유 안내
  │ → 나/ 카테고리는 품질 검사 면제
  │
  ▼
[3] 카테고리 자동 분류
  │ → "📁 [카테고리]로 분류했어요. 맞나요?"
  │
  ▼
[4] 저장 실행
  ├── wiki/source/ 에 원본 저장
  ├── wiki/wiki/[카테고리]/ 에 Wiki 페이지 생성
  ├── wiki/index.md 업데이트
  ├── wiki/log.md 기록
  └── git 커밋 (자동)
  │
  ▼
[5] 완료 보고
  "✅ 저장 완료: [제목]
   📂 카테고리: [카테고리]
   🔗 원본: [URL 또는 직접입력]
   ⚠️ 관련 페이지: [[링크1]] [[링크2]]
   📅 유통기한: YYYY-MM-DD"
```

## 선별 읽기 워크플로우 (질문/글쓰기)

```
요청
  │
  ▼
[1] wiki/index.md 전체 로드 (목차만 — 토큰 최소)
  │
  ▼
[2] 관련 페이지 선별 (최대 7~8개)
    기준: 키워드 매칭 + 나/ 카테고리 최소 2개 포함
  │
  ▼
[3] 선별된 페이지만 로드
  │
  ▼
[4] 답변 또는 글 초안 생성
    글쓰기 시: 외부자료로 팩트/구조 + 내 자료로 나다움
  │
  ▼
[5] 글쓰기 요청이면: writing_checklist.md 검수 적용
  │
  ▼
[6] 완성된 글도 수집 → wiki/wiki/콘텐츠/ 에 저장
```

## Wiki 페이지 표준 형식

모든 Wiki 페이지는 아래 frontmatter를 포함해야 한다:

```markdown
---
title: 페이지 제목
category: 카테고리명
tags: [태그1, 태그2]
source: URL 또는 직접입력
created: YYYY-MM-DD
expires: YYYY-MM-DD  # 나/ 카테고리는 never
type: url | text | pdf | image | diary | thought | experience | retrospect
---
```

## index.md 업데이트 규칙

새 페이지 저장 시 index.md를 아래 형식으로 업데이트:

```markdown
## 📁 [카테고리] (N)
- [[파일명]] - 한 줄 설명 #태그1 #태그2
```

헤더의 총 페이지 수와 날짜도 함께 업데이트.

## Git 자동 동기화 규칙

모든 작업 완료 후 반드시 아래 순서로 Git 커밋 & 푸시한다:

```bash
# 1. 변경사항 확인
git status

# 2. 전체 추가
git add .

# 3. 커밋 (작업 내용 요약)
git commit -m "feat: [작업 내용 한 줄 요약]"

# 4. 푸시
git push origin master
```

### 트리거 조건
- `수집` 완료 후 → 자동 git push
- 파일 생성/수정/삭제 후 → 자동 git push
- 작업 세션 종료 전 → 반드시 git push

### 커밋 메시지 규칙
- `feat:` 새 기능 추가
- `fix:` 오류 수정
- `docs:` 문서 수정
- `wiki:` Wiki 페이지 추가/수정

### Mac Mini 동기화
Mac Mini에서는 작업 시작 전 항상:
```bash
git pull origin master
```

## 기존 CLAUDE.md 기능 (유지)

이 파일은 LLM Wiki 오케스트레이터 전용이다.
글로벌 개발 전략(Phase 1~6)은 `~/.claude/CLAUDE.md`에 있다.

### 매일 마무리 루틴
사용자가 "오늘 끝", "오늘 마무리", "Day N 끝" 입력 시:
1. 블로그 포스팅 글 생성
2. 유튜브 스크립트 생성
3. 교육용 PPT (vanilla-slides HTML) 생성 → `ppt/` 에 저장
4. Notion 콘텐츠 아카이브 DB 저장
5. Git 커밋 & push
6. GitHub Pages URL 출력
