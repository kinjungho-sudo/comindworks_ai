## 2. 쉬운 비유

```
텔레그램 봇  → (제거) 클로드에게 직접 말하기
클로드       = 도서관 투입구 + 사서 역할
/source      = 원본 자료실 (Raw 데이터)
/wiki        = 사서가 정리한 책장 (카테고리별)
/guidelines  = 사서 운영 규칙서
index.md     = 도서관 목차 (7개만 골라 꺼내는 핵심)
log.md       = 업무일지 (세션 간 연속성)
노션 DB      = 유리창 진열대 (원본 열람용)
Obsidian     = 연결망 시각화 뷰어
```

## 3. 전체 아키텍처

```
[입력 - 클로드에게 직접]
  "수집 https://..."        ← URL
  "수집 [텍스트/일기]"      ← 메모/일기
  "수집 [파일 첨부]"        ← PDF/이미지
  "질문: ..."               ← 검색/활용
        │
        ▼
[입구 관리 - 품질 검사]
  - 중복 여부 확인
  - 품질 평가 (저품질 → 저장 거부 + 사유 안내)
  - 카테고리 자동 분류
        │
        ▼
[Claude Code 처리]
  - /source에 Raw 저장
  - Wiki 페이지 생성
  - 카테고리 폴더에 저장
  - index.md 업데이트
  - 노션 DB 저장
  - log.md 기록
  - Git 커밋
        │
        ▼
[완료 보고]
  "✅ 저장 완료: 제목
   📂 카테고리: AI에이전트
   🔗 노션: [링크]
   ⚠️ 관련 페이지: [[링크1]] [[링크2]]"
```

## 4. 입력 타입별 처리

| 입력 타입 | 명령어 예시 | 처리 방식 |
|----------|------------|----------|
| URL | `수집 https://...` | 스크래핑 → Wiki 생성 |
| 텍스트/일기 | `수집 오늘 느낀 것: ...` | 그대로 저장 → Wiki 생성 |
| PDF | `수집 [파일 첨부]` | 텍스트 추출 → Wiki 생성 |
| 이미지 | `수집 [이미지 첨부]` | OCR 추출 → Wiki 생성 |
| 질문 | `질문: n8n 관련 뭐 있어?` | 선별 읽기 → 답변 |
| 글쓰기 | `글쓰기: 블로그 주제 ...` | 선별 읽기 → 글 생성 |

## 7. 3가지 관리 체계

### ① 입구 관리 (저장 전 검사)

```
자료가 들어오면 Claude가 먼저 확인:

1. 중복 검사
   index.md에서 유사 페이지 검색
   → 80% 이상 유사 → "이미 유사한 페이지 있어요: [[링크]]
                       그래도 저장할까요?"

2. 품질 평가
   기준: 코마인드웍스 사업과 연관성 있는가?
         구체적인 내용이 있는가?
         단순 광고/스팸인가?
   → 저품질 → "품질 기준 미달이에요. 저장하지 않을게요."
   → 통과 → 다음 단계

3. 카테고리 자동 분류
   Claude가 내용 보고 카테고리 자동 판단
   → "📁 AI에이전트로 분류했어요. 맞나요?"
```

### ② 내부 관리 (저장된 후 관리)

```
1. 모순 처리
   새 페이지 저장 시 기존 페이지와 충돌 감지
   → "[[기존페이지]]와 내용이 충돌해요.
      어떤 걸 기준으로 할까요?"

2. 유통기한 태그
   페이지 생성 시 자동으로 날짜 태그 부여
   → expires: 2026-10-17 (6개월)
   → 점검 시 만료된 페이지 목록 리포트

3. 페이지 자동 분할
   페이지가 너무 커지면 (3,000자 초과)
   → "[[n8n-자동화]] 페이지가 너무 커졌어요.
      n8n-기초 / n8n-고급으로 분할할까요?"
```

### ③ 효율 관리 (대화 시작 시)

```
새 대화 시작하면 Claude가 자동 보고:

"📚 Wiki 현황 보고
 총 페이지: 42개
 이번 주 추가: 7개
 만료 예정: 3개
 모순 감지: 1건 (확인 필요)
 가장 많이 참조된 페이지: [[하네스엔지니어링]]

 오늘 뭘 할까요?"
```

## 8. 선별 읽기 워크플로우 (핵심 기능)

### 글쓰기 요청 시 동작 방식

```
나: "블로그 글 써줘. 주제: AI 에이전트로 업무 자동화"
        │
        ▼
1단계: index.md 전체 로드
       (목차만 읽음 - 토큰 최소)
        │
        ▼
2단계: 관련 페이지 선별 (7~8개)
       ✓ [[하네스엔지니어링]]
       ✓ [[일기-2026-04-15]] ← 내 감정
       ✓ [[경험-첫번째삽질]] ← 내 경험
       ✓ [[n8n-자동화]]
       ✓ [[생각-에이전트는팀원]] ← 내 생각
       ✓ [[플라이휠-전략]]
       ✓ [[유튜브-대본-원칙]]
        │
        ▼
3단계: 선별된 7개 페이지만 로드
        │
        ▼
4단계: 글 초안 작성
       - 외부 자료 5개로 팩트와 구조 잡기
       - 내 일기/경험/생각 3개로 '나다움' 입히기
        │
        ▼
5단계: 검수 체크리스트 (guidelines에서 로드)
       - 훅이 강한가?
       - 감정 곡선이 있는가?
       - 내 경험이 담겼는가?
        │
        ▼
6단계: 수정 후 최종본 완성
        │
        ▼
7단계: 완성된 글도 Wiki에 저장
       "수집 [완성된 글]" → 📁 콘텐츠에 저장
```

**결과**: 세상에 없는 나만의 글
